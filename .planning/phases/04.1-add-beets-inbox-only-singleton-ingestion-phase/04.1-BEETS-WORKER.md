# Phase 04.1 Beets Worker Runbook

## Automatic flow

/srv/media/inbox/slskd -> beets-inbox-watch.path -> beets-inbox-run.service -> in-place tag writes in inbox -> /srv/data/beets/reports + /srv/data/beets/unresolved

The watch is file-event driven from `/srv/media/inbox/slskd/*`, and the worker only runs singleton imports against inbox-scoped targets.

## Manual targeting

- `beets-inbox-runner /srv/media/inbox/slskd`
- `BEETS_DRY_RUN=1 beets-inbox-runner /srv/media/inbox/slskd`
- `beets-inbox-runner /srv/media/inbox/<selected-subpath>`

The runner enforces that supplied paths stay under `/srv/media/inbox` and rejects broader scan roots.

## Verification

- `bash tests/phase-04.1-beets-contract.sh`
- `just verify-phase-04.1`

Note: in this repository's `just` syntax, run `just verify-phase-04-1` (hyphen form); recipe comments preserve the canonical phase label `verify-phase-04.1`.

## Beatport authorization

The first interactive browser/paste-back flow for Beatport authorization must be completed once on the target host. Successful authorization stores tokens under `/srv/data/beets` and subsequent runs should not prompt again.

## Guardrails

- copy/move/link/hardlink remain disabled
- global /srv/media scans are forbidden
- SoundCloud stays deferred
