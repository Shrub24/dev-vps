# Phase 04.2: All-inbox Beets preprocessing with singleton auto-promotion - Context

**Gathered:** 2026-04-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Evolve the current Beets worker from slskd-only inbox tagging into an all-inbox native album-import and auto-promotion flow for `oci-melb-1`, so successful tracks move from `/srv/media/inbox` into `/srv/media/library` while hard failures and leftovers are demoted to `/srv/media/untagged`. This phase keeps `/srv/media` as the authoritative media root, `/srv/data/beets` as Beets state/report storage, and does not introduce a separate curation application or broader media-management scope.

</domain>

<decisions>
## Implementation Decisions

### Promotion Rules
- **D-01:** A track with a usable Beets match should be promoted by default even if some metadata is imperfect.
- **D-02:** Only true run failures block promotion in this phase. Missing or imperfect metadata alone is not a blocker.
- **D-03:** Imperfect-but-promoted tracks should be tracked in reports only; no extra review bucket or stricter promotion gate is introduced.
- **D-04:** Album import should rely on native Beets grouping/matching semantics (`group_albums: yes`) rather than custom worker-side grouping logic.

### Library Placement
- **D-05:** Promoted files land under `/srv/media/library` using a clean canonical library layout rather than preserving inbox/source folder identity.
- **D-06:** Top-level organization bias is flexible by match: use whichever of artist or label makes the release placement feel more natural for the catalog instead of forcing one universal artist-first or label-first rule.
- **D-07:** Files should still be promoted with simple fallback placement when release/album metadata is weak or incomplete.
- **D-08:** Filenames must not be renamed in this phase.

### Playback Visibility
- **D-09:** Navidrome should continue reading all of `/srv/media` during this phase rather than switching to `/srv/media/library` only.
- **D-10:** Failed inbox items remain playable through Navidrome; playback access must not depend on successful promotion.
- **D-11:** Short duplicate overlap during move/promotion is acceptable if the flow stays simple and reliable.
- **D-12:** Inbox remains a visible staging area, not the intended long-term listening surface.

### Failure Handling
- **D-13:** Hard-failing files and post-import leftovers are demoted out of inbox into `/srv/media/untagged` to prevent recursive reprocessing loops.
- **D-14:** Operator follow-up is report-driven first, using Beets outputs under `/srv/data/beets` rather than a new review UX.
- **D-15:** Repeated automatic failures may keep retrying and keep being logged; this phase does not add a parked/manual-reset mechanism.
- **D-16:** No extra manual review workflow is required beyond normal filesystem access plus reports.
- **D-17:** If any `.tmp` transfer file exists under inbox, worker execution exits without invoking Beets.
- **D-18:** Worker uses a fixed settle/debounce delay after transfer lock clears to batch near-simultaneous arrivals.
- **D-19:** Worker execution relies on native systemd single-instance behavior (`Type=oneshot`) rather than explicit runner-level lockfiles.

### the agent's Discretion
- Exact Beets `paths:` template details for artist-vs-label-biased placement, as long as filenames are preserved and source-folder identity is removed from final library paths.
- Exact definition of "true run failure" at the implementation level, as long as it stays narrower than metadata-quality concerns and supports sweep-to-untagged behavior.
- Exact report schema, filenames, and summary formatting under `/srv/data/beets`.
- Exact watcher/timer/backstop mechanics and sweep sequencing, as long as transfer-lock, settle delay, post-run demotion sweep, and single-run semantics remain true.

</decisions>

### Established Patterns
- `/srv/media` is the authoritative shared media root; service state stays under `/srv/data/*`.
- Path ownership and service-owned directories are managed declaratively with `systemd.tmpfiles.rules`.
- Media/service regressions are protected with shell contract tests rather than heavyweight test harnesses.
- Navidrome currently reads all of `/srv/media`, so inbox-vs-library visibility is governed by path layout rather than a second application boundary.

### Integration Points
- Expand Beets operation from `/srv/media/inbox/slskd` targeting to all-inbox targeting.
- Use native Beets album import and `paths:` placement for `/srv/media/library` without changing the higher-level `/srv/media` authority model.
- Keep reports and failure visibility rooted under `/srv/data/beets`.
- Update Beets and service-flow contract coverage to assert auto-promotion semantics and preserved playback visibility.

<specifics>
## Specific Ideas

- "Beets tagging shouldn't gate promotion... I want access to my music even if the tagger cant figure it out."
- Inbox should be temporary by default when the processing flow is working well.
- If manual curation becomes stricter later, the user still wants playback access and prefers filesystem move-based approval rather than an app workflow.
- Updated acceptance criteria require: PathModified watcher on inbox, `.tmp` lockout, settle delay, native quiet album import (`singletons: no`, `group_albums: yes`), demotion sweep to `/srv/media/untagged`, and zero-state inbox after successful runs.
- The catalog is underground-techno-heavy, so final placement may naturally fit `Artist/Release` or `Label/Release` depending on the match.

</specifics>
