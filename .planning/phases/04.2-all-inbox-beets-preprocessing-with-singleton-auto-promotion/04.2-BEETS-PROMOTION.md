# Phase 04.2 Beets Promotion Runbook

## Automatic flow

/srv/media/inbox/** -> beets-inbox-run.service -> /srv/media/library/<top-level>/<release>/<original filename> -> /srv/data/beets/reports + /srv/data/beets/unresolved

The singleton worker scans all inbox sources, processes one file at a time, promotes successful files, and keeps hard failures in place for report-first follow-up.

## Manual targeting

- `beets-inbox-runner /srv/media/inbox`
- `BEETS_DRY_RUN=1 beets-inbox-runner /srv/media/inbox`
- `beets-inbox-runner /srv/media/inbox/<selected-subpath>`

## Verification

- `bash tests/phase-04.2-beets-promotion-contract.sh`
- `just verify-phase-04.2`

Note: this repo executes `just verify-phase-04-2` due to the dotted-name workaround in `just` recipe names.

## Reports

- `/srv/data/beets/reports/<timestamp>-summary.txt`
- `/srv/data/beets/reports/<timestamp>-promoted-with-fallback.tsv`
- `/srv/data/beets/unresolved/<timestamp>-hard-failures.tsv`

## Guardrails

- filenames stay unchanged
- failed files remain in inbox
- Navidrome still reads /srv/media
- no dedicated review UI or parked retry flow
