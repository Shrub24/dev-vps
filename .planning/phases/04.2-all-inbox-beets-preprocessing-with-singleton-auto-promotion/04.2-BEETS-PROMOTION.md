# Phase 04.2 Beets Promotion Runbook

## Automatic flow

/srv/media/inbox/** (PathModified) -> beets-inbox-run.service -> transfer lock check (.tmp) -> settle delay -> quiet Beets album import (`singletons: no`, `group_albums: yes`) with built-in import log (`-l`) -> Beets `paths:` placement in /srv/media/library/<top-level>/<release>/<original filename> -> post-run sweep of leftover inbox audio -> /srv/media/untagged/**

The worker is transfer-safe and non-interactive: it exits early when active `.tmp` files are present, waits for a settle window, runs Beets headlessly in native album-import mode, then sweeps any audio still in inbox to `/srv/media/untagged`.

## Manual targeting

- `beets-inbox-runner /srv/media/inbox`
- `BEETS_DRY_RUN=1 beets-inbox-runner /srv/media/inbox`
- `beets-inbox-runner /srv/media/inbox/<selected-subpath>`

Optional runtime tuning:

- `BEETS_SETTLE_SECONDS=10`

## Verification

- `bash tests/phase-04.2-beets-promotion-contract.sh`
- `just verify-phase-04.2`

Note: this repo executes `just verify-phase-04-2` due to the dotted-name workaround in `just` recipe names.

## Import logs

- `/srv/data/beets/logs/<timestamp>-import.log`

## Guardrails

- filenames stay unchanged
- worker runs quiet/headless (`-q`) with album import (`singletons: no`) and Beets grouping (`group_albums: yes`)
- inbox processing is blocked while `.tmp` transfer files exist
- remaining inbox audio is demoted to `/srv/media/untagged`
- inbox returns to zero-state for eligible audio at end of run
- Navidrome still reads /srv/media
- no dedicated review UI or parked retry flow
